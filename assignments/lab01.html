Step 1: Set up your terminal environment

  1. For this course, you will develop software locally on your laptop. If you are not familiar with the Unix shell, please review my Shell Basics document.

  2. Terminal setup (Mac, Windows)

    For Mac:

    • Apple’s Terminal app should work ok
    • I prefer iTerm2 because it works well with my preferred terminal-mode editor, micro

    For Windows:

    • I recommend using Git For Windows. Git Bash offers a Unix-like shell environment.
    • If you already have Windows Subsystem for Linux, that’s also fine
  3. You may use the terminal-mode editor of your choice (e.g. micro, nano, vim, emacs)

Step 3: Set up your Go toolchain

  1. Install the Go toolchain. Be careful to choose the right build for your laptop (Mac, Windows)

    For Mac:

    • Newer Apple laptops with M1 or M2 processors need the darwin-arm64 build
    • Older Apple laptops with Intel processors need the darwin-amd64 build
    • The Go toolchain installs into /usr/local/ and is automatically added to the PATH (by installing itself into /etc/path.d/) 
        phil@Phils-MacBook-Pro:~ % which go
  /usr/local/go/bin/go

  phil@Phils-MacBook-Pro:~ % go version
  go version go1.20.5 darwin/arm64

    For Windows Git Bash:

    • Windows laptops need the windows-amd64 build
    • The Go toolchain installs into C:\Program Files\Go and is added to the PATH automatically 
        phil@PHILPETERSO43DB MINGW64 /
  $ which go
  /c/Program Files/Go/bin/go

  phil@PHILPETERSO43DB MINGW64 /
  $ go version
  go version go1.20.5 windows/amd64
  2. Install the dlv (delve) debugger. 
     go install github.com/go-delve/delve/cmd/dlv@latest

  3. Edit your shell config file (Mac, Windows)

    For Mac:

    1. Edit ~/.zshrc
    2. At the bottom of the file, add export PATH=~/go/bin:$PATH
    3. Apply the change to your shell: source ~/.zshrc

    For Windows Git Bash:

    1. Edit ~/.bashrc
    2. At the bottom of the file, add export PATH=~/go/bin:$PATH
    3. Apply the change to your shell: source ~/.bashrc
  4. Check to see that dlv is on the path 
     phil@Phils-MBP:~ % which dlv
 /Users/phil/go/bin/dlv
  5. Optional: If you want to use an Integrated Development Environment (IDE), you may use VS Code.
    1. VS Code will offer to install extensions for go and gopls. You should accept both.
  6. Unsupported: There is a Go-specific IDE called GoLand that you’re welcome to try. I won’t be supporting it.
