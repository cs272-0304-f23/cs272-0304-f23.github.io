Project02

Due: Tue, Oct 3, 2023 at 11:59 PM to Github Classroom Assignment

Requirements

  1. For this project, you will extend the functionality of the MVP search engine you build in project01
  2. You will apply these Information Retrieval (IR) techniques as discussed in lecture.
    1. Words will again be stemmed using the Snowball stemmer
    2. Your inverted index will omit Stop Words using this list
    3. You will rank search results using the TF-IDF technique
  3. You will reuse these elements of prior assignments:
    1. the corpus of 10 books you built for lab04
    2. the crawler and inverted index you built for project01
  4. Architectural requirements:
    1. Indexing and searching must be separated in your code so that your web server runs while crawling is working
    2. Your local web server (on localhost or 127.0.0.1) will serve the top10 corpus for your local crawler
    3. You must not use global variables. Make at least one element of your code use methods in an object-oriented style (hint: an index might be helpful)

Given

  1. From previous assignments:
    1. Snowball stemmer
    2. Corpus of 10 books from Project Gutenberg
  2. Lecture material on scoring and ranking search results
  3. You can use the Go sort.Interface interface to do the sorting
  4. You can use the stopwords-iso collection of English stopwords. The json list is formatted similarly to Go.

Required Test Cases

  1. Data structures for test cases are provided:
    • Stopword tests
    • TF-IDF tests
  2. You’ll see that I called the lab04 corpus “top10”. You can rename that if you like, but don’t change the wanted URLs or scores.

Implementation Tips

  1. You’ll want to use the Go float64 type to calculate the TF-IDF score. For example, you might calculate 
     tf := (float64) termCount / (float64) wordsInDoc
  2. Some terms generate the same TF-IDF score. In order to make the test cases deterministic (same order for all hits with the same score), your implementation of the Less() function in sort.Interface should check for the same score, and if so, also sort by the URL. For example: 
     if scoreOfI == scoreOfJ {
     return urlOfJ > urlOfI
 }
